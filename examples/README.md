# Reusable Examples

These examples are starting points with explicit assumptions, not copy-paste production content.

- [Search cookbook](search-cookbook.md): common analyst, security, reliability, and administration queries.
- [Configuration patterns](configuration-patterns.md): app layout, input, parsing, and retention review examples.
- [Test cases](test-driven-spl.md): use `makeresults` to test SPL logic before real data.

Before using an example:

1. Replace indexes and sourcetypes with authorized values.
2. Confirm fields, types, semantics, time zone, and event unit.
3. Bound time and expected cardinality.
4. Test positive, negative, boundary, null, duplicate, and late-data cases.
5. Inspect performance in representative conditions.
6. Document ownership and what the result does not prove.
